Primitive Data Types

Java has eight primitive data types that store simple values directly in memory.

Common Primitive Types 
// Integer types
int age = 25;              // 32-bit integer
long population = 8000000000L;  // 64-bit integer (note the L)

// Floating-point types
double price = 29.99;      // 64-bit decimal
float percentage = 0.85f;  // 32-bit decimal (note the f)

// Boolean type
boolean isActive = true;
boolean hasPermission = false;

// Character type
char grade = 'A';
char symbol = '$';
Memory Tip: Primitive types store actual values, not references. They're faster and use less memory than reference types.

Reference Data Types

Reference types store references (addresses) to objects in memory. The most common is String.

String Reference Type 
// String is a reference type
String name = "John Doe";
String email = null;  // Can be null
String empty = "";    // Empty string is different from null

// Other reference types
Integer boxedAge = 25;        // Wrapper class for int
Boolean boxedFlag = true;     // Wrapper class for boolean

Variable Declaration & Initialization

Variables must be declared with a type and can be initialized with a value.

Variable Declaration Patterns 
public class VariableDemo {
    // Class-level variables (fields)
    private int classField = 100;
    private String className;  // Initialized to null by default
    
    public void demonstrateVariables() {
        // Local variables - must be initialized before use
        int localVar;           // Declared but not initialized
        // System.out.println(localVar); // ERROR! Must initialize first
        
        localVar = 50;          // Now initialized
        System.out.println(localVar); // OK
        
        // Declaration with initialization
        String message = "Hello World";
        boolean isReady = true;
        
        // Multiple declarations
        int x = 1, y = 2, z = 3;
        String firstName = "John", lastName = "Doe";
    }
}

Variable Scope

Variable scope determines where variables can be accessed in your code.

Understanding Variable Scope 
public class ScopeDemo {
    // Class scope - accessible throughout the class
    private int classVariable = 10;
    
    public void methodOne() {
        // Method scope - only accessible within this method
        int methodVariable = 20;
        
        if (classVariable > 5) {
            // Block scope - only accessible within this if block
            int blockVariable = 30;
            
            System.out.println(classVariable);  // OK - class scope
            System.out.println(methodVariable); // OK - method scope
            System.out.println(blockVariable);  // OK - current block
        }
        
        // System.out.println(blockVariable); // ERROR! Out of scope
    }
    
    public void methodTwo() {
        System.out.println(classVariable);  // OK - class scope
        // System.out.println(methodVariable); // ERROR! Different method
    }
}
Scope Rule: Variables are only accessible within the scope where they're declared. Local variables must be initialized before use.

Type Casting and Conversion

Sometimes you need to convert between different data types.

Type Casting Examples 
public class TypeCastingDemo {
    public void demonstrateCasting() {
        // Automatic casting (widening) - no data loss
        int intValue = 100;
        long longValue = intValue;     // int to long - automatic
        double doubleValue = intValue; // int to double - automatic
        
        // Manual casting (narrowing) - potential data loss
        double largeDouble = 99.99;
        int truncatedInt = (int) largeDouble; // 99 (decimal part lost)
        
        // String to number conversion
        String numberString = "42";
        int convertedInt = Integer.parseInt(numberString);
        double convertedDouble = Double.parseDouble("3.14");
        
        // Number to String conversion
        int age = 25;
        String ageString = String.valueOf(age);
        String formattedAge = "" + age; // Quick conversion trick
        
        System.out.println("Converted int: " + convertedInt);
        System.out.println("Age as string: " + ageString);
    }
}

Best Practices for Variables

Use descriptive names: userName instead of u
Initialize variables: Always initialize local variables before use
Choose appropriate types: Use int for whole numbers, String for text
Minimize scope: Declare variables in the smallest scope needed

Key Takeaways

Primitive vs Reference: Primitives store values directly, references store memory addresses
Variable Scope: Variables are only accessible within their declared scope
Type Casting: Convert between types carefully to avoid data loss
Initialization: Local variables must be initialized before use
