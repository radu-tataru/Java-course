1. What are Enums?

An enum (enumeration) is a special data type that represents a fixed set of constants. Enums provide type safety and make code more readable.

Why Use Enums?

  • Type Safety: Prevent invalid values at compile time
  • Readability: Self-documenting code with meaningful names
  • Maintainability: Constants grouped in one place
  • Switch Compatibility: Work seamlessly with switch statements
  • IDE Support: Auto-completion and refactoring support

Before Enums (String/Int Constants)

// ❌ Problems: No type safety, magic strings
public class OrderOld {
    public static final String STATUS_PENDING = "PENDING";
    public static final String STATUS_SHIPPED = "SHIPPED";
    public static final String STATUS_DELIVERED = "DELIVERED";

    private String status;

    public void setStatus(String status) {
        this.status = status;  // Can be ANY string!
    }
}

// Usage - error-prone
order.setStatus("SHIPPED");
order.setStatus("shipped");   // Typo - runtime bug!
order.setStatus("INVALID");   // Compiles but wrong!

With Enums (Type Safe)

// ✅ Type-safe, compile-time checking
public enum OrderStatus {
    PENDING, SHIPPED, DELIVERED
}

public class Order {
    private OrderStatus status;

    public void setStatus(OrderStatus status) {
        this.status = status;  // Only valid enum values allowed
    }
}

// Usage - type safe
order.setStatus(OrderStatus.SHIPPED);   // ✅ Valid
// order.setStatus("shipped");  // ❌ Compile error!
Note: Enums are implicitly final and extend java.lang.Enum. Each enum constant is a public static final instance of the enum type.

2. Creating and Using Enums

Basic Enum

public enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}

// Usage
Day today = Day.MONDAY;
System.out.println("Today is: " + today);  // Today is: MONDAY

// Comparison
if (today == Day.FRIDAY) {
    System.out.println("Weekend is near!");
}

Enum in Switch Statement

public static void printDayType(Day day) {
    switch (day) {
        case SATURDAY:
        case SUNDAY:
            System.out.println("Weekend");
            break;
        case MONDAY:
        case TUESDAY:
        case WEDNESDAY:
        case THURSDAY:
        case FRIDAY:
            System.out.println("Weekday");
            break;
    }
}

// Usage
printDayType(Day.SATURDAY);  // Weekend
printDayType(Day.MONDAY);    // Weekday

Built-in Enum Methods

// values() - get all enum constants
Day[] allDays = Day.values();
for (Day day : allDays) {
    System.out.println(day);
}

// valueOf() - convert string to enum
Day day1 = Day.valueOf("MONDAY");
System.out.println(day1);  // MONDAY

// ordinal() - get position (0-indexed)
System.out.println(Day.MONDAY.ordinal());    // 0
System.out.println(Day.FRIDAY.ordinal());    // 4

// name() - get constant name as string
System.out.println(Day.TUESDAY.name());      // TUESDAY

3. Enums with Fields and Methods

Enum with Constructor and Fields

public enum Planet {
    MERCURY(3.303e+23, 2.4397e6),
    VENUS(4.869e+24, 6.0518e6),
    EARTH(5.976e+24, 6.37814e6),
    MARS(6.421e+23, 3.3972e6);

    private final double mass;    // in kilograms
    private final double radius;  // in meters

    // Constructor (must be private or package-private)
    Planet(double mass, double radius) {
        this.mass = mass;
        this.radius = radius;
    }

    public double getMass() {
        return mass;
    }

    public double getRadius() {
        return radius;
    }

    // Calculate surface gravity
    public double surfaceGravity() {
        final double G = 6.67300E-11;
        return G * mass / (radius * radius);
    }
}

// Usage
Planet earth = Planet.EARTH;
System.out.println("Earth's mass: " + earth.getMass());
System.out.println("Surface gravity: " + earth.surfaceGravity());

Enum with Methods

public enum Operation {
    PLUS("+") {
        public double apply(double x, double y) { return x + y; }
    },
    MINUS("-") {
        public double apply(double x, double y) { return x - y; }
    },
    TIMES("*") {
        public double apply(double x, double y) { return x * y; }
    },
    DIVIDE("/") {
        public double apply(double x, double y) { return x / y; }
    };

    private final String symbol;

    Operation(String symbol) {
        this.symbol = symbol;
    }

    public String getSymbol() {
        return symbol;
    }

    // Abstract method - each constant must implement
    public abstract double apply(double x, double y);
}

// Usage
double result = Operation.PLUS.apply(5, 3);
System.out.println("5 + 3 = " + result);  // 8.0

for (Operation op : Operation.values()) {
    System.out.println(op.getSymbol() + ": " + op.apply(10, 5));
}
// Output:
// +: 15.0
// -: 5.0
// *: 50.0
// /: 2.0

4. Real-World Enum Examples

Example 1: HTTP Status Codes

public enum HttpStatus {
    OK(200, "OK"),
    CREATED(201, "Created"),
    BAD_REQUEST(400, "Bad Request"),
    UNAUTHORIZED(401, "Unauthorized"),
    NOT_FOUND(404, "Not Found"),
    INTERNAL_SERVER_ERROR(500, "Internal Server Error");

    private final int code;
    private final String message;

    HttpStatus(int code, String message) {
        this.code = code;
        this.message = message;
    }

    public int getCode() {
        return code;
    }

    public String getMessage() {
        return message;
    }

    // Find status by code
    public static HttpStatus fromCode(int code) {
        for (HttpStatus status : values()) {
            if (status.code == code) {
                return status;
            }
        }
        throw new IllegalArgumentException("Unknown status code: " + code);
    }
}

// Usage
HttpStatus status = HttpStatus.OK;
System.out.println(status.getCode() + " " + status.getMessage());  // 200 OK

HttpStatus notFound = HttpStatus.fromCode(404);
System.out.println(notFound);  // NOT_FOUND

Example 3: User Roles

public enum UserRole {
    GUEST(0, "Guest User"),
    USER(1, "Regular User"),
    MODERATOR(2, "Moderator"),
    ADMIN(3, "Administrator");

    private final int level;
    private final String displayName;

    UserRole(int level, String displayName) {
        this.level = level;
        this.displayName = displayName;
    }

    public int getLevel() {
        return level;
    }

    public String getDisplayName() {
        return displayName;
    }

    public boolean hasPermission(UserRole required) {
        return this.level >= required.level;
    }
}

// Usage
public class User {
    private UserRole role;

    public boolean canModerate() {
        return role.hasPermission(UserRole.MODERATOR);
    }

    public boolean canAdminister() {
        return role.hasPermission(UserRole.ADMIN);
    }
}

User user = new User();
user.setRole(UserRole.MODERATOR);
System.out.println(user.canModerate());     // true
System.out.println(user.canAdminister());   // false

5. Enum Collections and Utilities

EnumSet - Specialized Set for Enums

import java.util.EnumSet;

public enum Color {
    RED, GREEN, BLUE, YELLOW, ORANGE, PURPLE
}

// Create EnumSet
EnumSet primaryColors = EnumSet.of(Color.RED, Color.GREEN, Color.BLUE);
EnumSet warmColors = EnumSet.of(Color.RED, Color.ORANGE, Color.YELLOW);
EnumSet allColors = EnumSet.allOf(Color.class);

System.out.println("Primary: " + primaryColors);
System.out.println("Warm: " + warmColors);

// Set operations
EnumSet redColors = EnumSet.of(Color.RED, Color.ORANGE);
System.out.println(warmColors.contains(Color.RED));  // true

EnumMap - Specialized Map for Enums

import java.util.EnumMap;

public class PriceList {
    private EnumMap prices = new EnumMap<>(Product.class);

    public void setPrice(Product product, double price) {
        prices.put(product, price);
    }

    public double getPrice(Product product) {
        return prices.getOrDefault(product, 0.0);
    }
}

enum Product {
    LAPTOP, PHONE, TABLET
}

// Usage
PriceList priceList = new PriceList();
priceList.setPrice(Product.LAPTOP, 999.99);
priceList.setPrice(Product.PHONE, 599.99);

System.out.println("Laptop: $" + priceList.getPrice(Product.LAPTOP));

Practice Exercises

Exercise 1: Traffic Light
Create a TrafficLight enum with RED, YELLOW, GREEN. Add a method to get the next light.
Show Solution 
public enum TrafficLight {
    RED("Stop", 30),
    YELLOW("Prepare to stop", 5),
    GREEN("Go", 45);

    private final String action;
    private final int duration;

    TrafficLight(String action, int duration) {
        this.action = action;
        this.duration = duration;
    }

    public String getAction() {
        return action;
    }

    public int getDuration() {
        return duration;
    }

    public TrafficLight next() {
        switch (this) {
            case RED:
                return GREEN;
            case GREEN:
                return YELLOW;
            case YELLOW:
                return RED;
            default:
                throw new IllegalStateException();
        }
    }
}

// Usage
TrafficLight light = TrafficLight.RED;
System.out.println("Current: " + light + " - " + light.getAction());

light = light.next();
System.out.println("Next: " + light + " - " + light.getAction());
// Output:
// Current: RED - Stop
// Next: GREEN - Go
Exercise 2: Card Deck
Create Suit and Rank enums for playing cards.
Show Solution 
public enum Suit {
    CLUBS("♣"), DIAMONDS("♦"), HEARTS("♥"), SPADES("♠");

    private final String symbol;

    Suit(String symbol) {
        this.symbol = symbol;
    }

    public String getSymbol() {
        return symbol;
    }
}

public enum Rank {
    TWO(2), THREE(3), FOUR(4), FIVE(5), SIX(6), SEVEN(7), EIGHT(8),
    NINE(9), TEN(10), JACK(10), QUEEN(10), KING(10), ACE(11);

    private final int value;

    Rank(int value) {
        this.value = value;
    }

    public int getValue() {
        return value;
    }
}

public class Card {
    private final Rank rank;
    private final Suit suit;

    public Card(Rank rank, Suit suit) {
        this.rank = rank;
        this.suit = suit;
    }

    @Override
    public String toString() {
        return rank + " of " + suit + " " + suit.getSymbol();
    }
}

// Usage
Card card = new Card(Rank.ACE, Suit.SPADES);
System.out.println(card);  // ACE of SPADES ♠

Key Takeaways

Type Safety: Enums provide compile-time type checking for constant values.
Fixed Set: Enums represent a fixed set of constants known at compile time.
Rich Enums: Can have fields, constructors, and methods for complex behavior.
Switch Compatible: Work seamlessly with switch statements for clean code.
Built-in Methods: Use values(), valueOf(), ordinal(), and name() for enum operations.
EnumSet/EnumMap: Specialized high-performance collections for enums.
Singleton by Nature: Each enum constant is a singleton instance.
