Java Packages

A package in Java is a namespace that organizes classes and interfaces. It helps avoid name conflicts and makes code easier to manage.

Best Practice: Always organize your Java classes in logical packages, especially for larger projects.

Refactored Vehicle Class with Access Modifiers

Here's our Vehicle class properly implemented with access modifiers and encapsulation:

public class Vehicle {
    private static String marca = "Dacia";
    private String type;
    private int horsepower;
    private String comment;

    private static int count = 0;

    public Vehicle(String type, int horsepower, String comment){
        this.type = type;
        this.horsepower = horsepower;
        this.comment = comment;
        count++;
    }

    public void honk(){
        System.out.println(this.type + " " + marca + " a claxonat!");
    }

    public static void sum(){
        System.out.println(count);
    }

    // Getter methods
    public String getType() {
        return type;
    }

    public int getHorsepower() {
        return horsepower;
    }

    public String getComment() {
        return comment;
    }

    public static String getMarca() {
        return marca;
    }

    public static int getCount() {
        return count;
    }

    // Setter methods
    public void setType(String type) {
        this.type = type;
    }

    public void setHorsepower(int horsepower) {
        this.horsepower = horsepower;
    }

    public void setComment(String comment) {
        this.comment = comment;
    }

    public static void setMarca(String marca) {
        Vehicle.marca = marca;
    }
}

Why Use Getters and Setters?

Control Access and Behavior: Validate or restrict changes to fields.
Flexibility for Future Changes: Internal implementation can change without affecting external code.
Debugging and Logging: Add logs or breakpoints inside access methods.
Protect Invariants and Business Rules: Enforce rules like non-negative horsepower.
Important: Always make fields private and provide public getter/setter methods when external access is needed. This is a fundamental principle of encapsulation.

The Four Pillars of OOP

1. Encapsulation

The practice of hiding internal data and only exposing access through public methods, typically using getters and setters.

Example: Private fields with public accessor methods

2. Inheritance

A mechanism that allows one class to acquire the properties and behaviors (fields and methods) of another class.

Example: class Car extends Vehicle

3. Polymorphism

The ability of different classes to respond to the same method call in different ways, often through method overriding or overloading.

Example: Same method name, different implementations

4. Abstraction

The process of hiding complex implementation details and showing only the essential features of an object.

Example: Abstract classes and interfaces
