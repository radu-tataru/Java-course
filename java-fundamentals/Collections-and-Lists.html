Arrays vs Collections: When to Use What?

Before diving into collections, let's understand when to use arrays versus collections.

Feature Arrays Collections (List)
Size ❌ Fixed size ✅ Dynamic (grows/shrinks)
Performance ✅ Faster for large datasets ⚠️ Slight overhead
Ease of Use ❌ Manual size management ✅ Rich set of methods
Type Safety ✅ Built-in ✅ With generics
Memory ✅ Lower overhead ❌ Higher overhead
Rule of Thumb: Use arrays for fixed-size, performance-critical data. Use collections for dynamic data that changes size during runtime.

Generic Types: Type Safety Made Easy

Generics allow you to specify what type of objects a collection can hold, providing compile-time type safety.

Without Generics (Old Way - Don't Do This)

// Without generics - not recommended
List list = new ArrayList();
list.add("Hello");
list.add(42);        // Oops! Mixed types
list.add("World");

// Requires casting and can cause runtime errors
String first = (String) list.get(0);  // Cast required
String second = (String) list.get(1); // Runtime error! 42 is not a String

With Generics (Modern Way)

// With generics - recommended
List<String> stringList = new ArrayList<>();
stringList.add("Hello");
stringList.add("World");
// stringList.add(42);  // Compile error! Only Strings allowed

// No casting needed
String first = stringList.get(0);  // Safe and clean
String second = stringList.get(1); // No casting required
Diamond Operator: Since Java 7, you can use the diamond operator <> on the right side to avoid repeating the generic type.

ArrayList: Your Go-To List Implementation

ArrayList is the most commonly used List implementation. It's backed by a dynamic array that grows as needed.

Creating and Populating Lists

import java.util.ArrayList;
import java.util.List;

public class ListExamples {
    public static void main(String[] args) {
        // Create an empty list
        List<String> fruits = new ArrayList<>();

        // Add elements
        fruits.add("Apple");
        fruits.add("Banana");
        fruits.add("Cherry");

        // Add at specific position
        fruits.add(1, "Apricot");  // Insert at index 1

        // Create list with initial values (Java 9+)
        List<String> colors = List.of("Red", "Green", "Blue");

        // Note: List.of() creates immutable list
        // colors.add("Yellow"); // Would throw exception

        // For mutable list with initial values
        List<String> mutableColors = new ArrayList<>();
        mutableColors.add("Red");
        mutableColors.add("Green");
        mutableColors.add("Blue");
    }
}

Common List Operations

Essential Methods

List<String> animals = new ArrayList<>();

// Adding elements
animals.add("Cat");           // Add to end
animals.add("Dog");
animals.add(0, "Bird");       // Add at specific index

// Accessing elements
String firstAnimal = animals.get(0);    // Get element at index
int size = animals.size();               // Get number of elements
boolean isEmpty = animals.isEmpty();     // Check if empty

// Checking contents
boolean hasCat = animals.contains("Cat");           // Check if contains element
int catIndex = animals.indexOf("Cat");              // Find index of element (-1 if not found)

// Modifying elements
animals.set(1, "Hamster");              // Replace element at index
animals.remove("Dog");                  // Remove by value
animals.remove(0);                      // Remove by index

// Clearing
animals.clear();                        // Remove all elements

System.out.println("Animals: " + animals);  // ArrayList has nice toString()
Method Purpose Example
add(element) Add to end list.add("item")
add(index, element) Insert at position list.add(0, "first")
get(index) Access element String item = list.get(0)
size() Get length int count = list.size()
contains(element) Check existence if (list.contains("item"))
remove(element) Remove by value list.remove("item")

Enhanced For-Loop (For-Each)

The enhanced for-loop (also called for-each loop) is the most elegant way to iterate through collections.

Different Ways to Loop Through Lists

List<String> languages = List.of("Java", "Python", "JavaScript", "Go");

// 1. Enhanced for-loop (for-each) - RECOMMENDED
System.out.println("=== Enhanced For-Loop ===");
for (String language : languages) {
    System.out.println("Language: " + language);
}

// 2. Traditional for-loop with index
System.out.println("=== Traditional For-Loop ===");
for (int i = 0; i < languages.size(); i++) {
    String language = languages.get(i);
    System.out.println("Language " + i + ": " + language);
}

// 3. While loop
System.out.println("=== While Loop ===");
int index = 0;
while (index < languages.size()) {
    System.out.println("Language: " + languages.get(index));
    index++;
}

// 4. Iterator (advanced)
System.out.println("=== Iterator ===");
for (Iterator<String> it = languages.iterator(); it.hasNext(); ) {
    String language = it.next();
    System.out.println("Language: " + language);
}
Best Practice: Use enhanced for-loop unless you need the index. It's cleaner, less error-prone, and works with all collections.

When to Use Which Loop

Enhanced for-loop
When you just need the values (most common case)
Traditional for-loop
When you need the index or want to modify the list during iteration
Iterator
When you need to safely remove elements during iteration

Practical Example: Link Management

Here's how List<String> is used in real projects, specifically in the file reading project:

import java.util.ArrayList;
import java.util.List;

public class LinkManager {

    // Method that returns a list of links from a file
    public List<String> readLinksFromFile(String fileName) {
        List<String> links = new ArrayList<>();

        // Simulate reading from different file types
        if (fileName.endsWith(".txt")) {
            links.add("https://www.google.com");
            links.add("https://www.github.com");
        } else if (fileName.endsWith(".json")) {
            links.add("https://www.stackoverflow.com");
            links.add("https://www.oracle.com");
        }

        return links;  // Return the entire list
    }

    // Method that processes a list of links
    public void processLinks(List<String> links) {
        System.out.println("Processing " + links.size() + " links:");

        // Enhanced for-loop to process each link
        for (String link : links) {
            if (isValidUrl(link)) {
                System.out.println("✅ Valid: " + link);
            } else {
                System.out.println("❌ Invalid: " + link);
            }
        }
    }

    // Filter links by domain
    public List<String> filterByDomain(List<String> links, String domain) {
        List<String> filtered = new ArrayList<>();

        for (String link : links) {
            if (link.contains(domain)) {
                filtered.add(link);
            }
        }

        return filtered;
    }

    private boolean isValidUrl(String url) {
        return url.startsWith("http://") || url.startsWith("https://");
    }

    public static void main(String[] args) {
        LinkManager manager = new LinkManager();

        // Read links from file
        List<String> allLinks = manager.readLinksFromFile("links.txt");

        // Process all links
        manager.processLinks(allLinks);

        // Filter for specific domain
        List<String> githubLinks = manager.filterByDomain(allLinks, "github");
        System.out.println("GitHub links: " + githubLinks);
    }
}
Project Connection: This exact pattern is used in Project Features Step 3, where each file reader returns a List<String> of links!

Performance Considerations

Operation ArrayList When to Worry
add() at end ✅ O(1) - Very fast Never
get(index) ✅ O(1) - Very fast Never
add(index, element) ⚠️ O(n) - Slower Large lists, frequent inserts
remove(index) ⚠️ O(n) - Slower Large lists, frequent removals
contains() ⚠️ O(n) - Slower Large lists, frequent searches
For Beginners: Don't worry about performance until you have thousands of elements. ArrayList is perfectly fine for most applications.

Common Pitfalls and How to Avoid Them

1. Index Out of Bounds

List<String> items = new ArrayList<>();
items.add("First");

// ❌ WRONG - IndexOutOfBoundsException
// String second = items.get(1);  // No element at index 1!

// ✅ CORRECT - Check size first
if (items.size() > 1) {
    String second = items.get(1);
} else {
    System.out.println("List doesn't have a second element");
}

2. Modifying List During Iteration

List<String> numbers = new ArrayList<>();
numbers.add("1");
numbers.add("2");
numbers.add("3");

// ❌ WRONG - ConcurrentModificationException
// for (String num : numbers) {
//     if (num.equals("2")) {
//         numbers.remove(num);  // Don't modify during for-each!
//     }
// }

// ✅ CORRECT - Use traditional loop backwards
for (int i = numbers.size() - 1; i >= 0; i--) {
    if (numbers.get(i).equals("2")) {
        numbers.remove(i);
    }
}

3. Null Pointer Exceptions

List<String> items = null;

// ❌ WRONG - NullPointerException
// items.add("Hello");

// ✅ CORRECT - Initialize the list
items = new ArrayList<>();
items.add("Hello");

// ✅ ALSO CORRECT - Check for null
if (items != null) {
    items.add("Hello");
}
