Arithmetic Operators

Arithmetic operators perform mathematical calculations on numeric values.

Operator Name Example Result
+ Addition 5 + 3 8
- Subtraction 10 - 4 6
* Multiplication 6 * 7 42
/ Division 20 / 4 5
% Modulus (Remainder) 17 % 5 2
++ Increment x++ or ++x Increases by 1
-- Decrement x-- or --x Decreases by 1

Arithmetic Examples

int a = 10;
int b = 3;

int sum = a + b;          // 13
int difference = a - b;   // 7
int product = a * b;      // 30
int quotient = a / b;     // 3 (integer division)
int remainder = a % b;    // 1

// Increment and Decrement
int x = 5;
x++;  // x is now 6 (post-increment)
++x;  // x is now 7 (pre-increment)
x--;  // x is now 6 (post-decrement)
--x;  // x is now 5 (pre-decrement)
Integer Division: When dividing two integers, the result is truncated (no decimal). Example: 10 / 3 = 3 (not 3.333...). Use double for decimal results: 10.0 / 3 = 3.333...

Comparison (Relational) Operators

Comparison operators compare two values and return a boolean result (true or false).

Operator Name Example Returns true when...
== Equal to 5 == 5 Values are equal
!= Not equal to 5 != 3 Values are not equal
> Greater than 8 > 5 Left is greater
< Less than 3 < 7 Left is smaller
>= Greater than or equal 5 >= 5 Left is greater or equal
<= Less than or equal 4 <= 9 Left is smaller or equal

Comparison Examples

int age = 25;
int votingAge = 18;

boolean canVote = age >= votingAge;  // true
boolean isTeen = age >= 13 && age <= 19;  // false
boolean isAdult = age >= 18;  // true

// String comparison (use .equals() for Strings!)
String name1 = "Alice";
String name2 = "Bob";
boolean sameNam = name1.equals(name2);  // false (correct)
// boolean wrong = name1 == name2;  // DON'T use == for Strings!
Common Mistake: Use .equals() to compare Strings, not ==. The == operator compares object references, not content! 
String a = "Hello";
String b = "Hello";
a.equals(b)  // ✅ Correct: true
a == b       // ❌ Wrong: compares references

Logical Operators

Logical operators combine multiple boolean expressions.

Operator Name Example Description
&& Logical AND x > 5 && x < 10 True if BOTH conditions are true
|| Logical OR x < 5 || x > 10 True if AT LEAST ONE condition is true
! Logical NOT !(x == 5) Reverses the boolean value

Logical Operator Examples

int temperature = 75;
boolean isSunny = true;

// AND: Both conditions must be true
boolean perfectWeather = temperature > 70 && temperature < 80 && isSunny;  // true

// OR: At least one condition must be true
boolean stayInside = temperature < 50 || temperature > 90 || !isSunny;  // false

// NOT: Reverses the boolean
boolean isRaining = !isSunny;  // false

// Complex conditions
int score = 85;
boolean passedWithDistinction = score >= 80 && score <= 100;  // true
boolean needsRetake = score < 60 || score > 100;  // false

Assignment Operators

Assignment operators assign values to variables with optional arithmetic operations.

Operator Example Equivalent to
=x = 5Assigns 5 to x
+=x += 3x = x + 3
-=x -= 2x = x - 2
*=x *= 4x = x * 4
/=x /= 2x = x / 2
%=x %= 3x = x % 3
 
int count = 10;
count += 5;   // count is now 15
count -= 3;   // count is now 12
count *= 2;   // count is now 24
count /= 4;   // count is now 6
count %= 5;   // count is now 1

Ternary Operator (Conditional)

The ternary operator is a shorthand for simple if-else statements.

Syntax: condition ? valueIfTrue : valueIfFalse 
// Without ternary (verbose)
int age = 20;
String status;
if (age >= 18) {
    status = "Adult";
} else {
    status = "Minor";
}

// With ternary (concise)
String status = (age >= 18) ? "Adult" : "Minor";

// More examples
int a = 10, b = 20;
int max = (a > b) ? a : b;  // max = 20

int score = 75;
String grade = (score >= 60) ? "Pass" : "Fail";  // grade = "Pass"

boolean hasDiscount = (age > 65 || age < 12) ? true : false;
Use Case: Ternary operators are perfect for simple assignments based on a condition. They make code more concise and readable for straightforward conditions.

Operator Precedence

When multiple operators appear in an expression, Java follows a specific order of evaluation (precedence).

Priority Operators Type
1 (Highest)++ -- !Unary
2* / %Multiplicative
3+ -Additive
4< <= > >=Relational
5== !=Equality
6&&Logical AND
7||Logical OR
8 (Lowest)?:Ternary
9= += -= *= /= %=Assignment
 
// Precedence examples
int result = 5 + 3 * 2;     // 11 (not 16) - multiplication first
int result2 = (5 + 3) * 2;  // 16 - parentheses override precedence

boolean check = 10 > 5 && 3 < 7;  // true (comparison before logical)
boolean check2 = !false && true;   // true (NOT before AND)

// Use parentheses for clarity!
int complex = ((a + b) * c) / d;  // Clear and explicit
Best Practice: When in doubt, use parentheses () to make your intentions explicit. It improves code readability and prevents bugs from unexpected precedence.

Practice Exercise

Try This: Create a simple calculator program using different operators. 
public class OperatorsPractice {
    public static void main(String[] args) {
        // Arithmetic practice
        int a = 15, b = 4;
        System.out.println("Sum: " + (a + b));
        System.out.println("Difference: " + (a - b));
        System.out.println("Product: " + (a * b));
        System.out.println("Quotient: " + (a / b));
        System.out.println("Remainder: " + (a % b));

        // Comparison practice
        System.out.println("Is a greater than b? " + (a > b));
        System.out.println("Is a equal to b? " + (a == b));

        // Logical practice
        boolean isPositive = a > 0;
        boolean isEven = a % 2 == 0;
        System.out.println("Is positive AND even? " + (isPositive && isEven));

        // Ternary practice
        String result = (a > b) ? "a is larger" : "b is larger";
        System.out.println(result);
    }
}
