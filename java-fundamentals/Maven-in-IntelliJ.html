1. What is Maven?

Apache Maven is a powerful build automation and project management tool for Java applications. It handles dependency management, project building, testing, and deployment.

Why Use Maven?

  • Dependency Management: Automatically download and manage external libraries (JUnit, Selenium, Apache POI, etc.)
  • Standardized Project Structure: Consistent folder organization across all Maven projects
  • Build Automation: Compile, test, package, and deploy with simple commands
  • Plugin Ecosystem: Extend functionality with thousands of plugins
  • Multi-Module Projects: Manage complex projects with multiple sub-modules
Maven vs Manual JAR Management: Instead of manually downloading JAR files and adding them to your project, Maven automatically fetches dependencies from central repositories and manages version conflicts.

2. Creating a Maven Project in IntelliJ

Method 1: New Project Wizard

  1. Open IntelliJ IDEA
  2. Click File → New → Project
  3. Select Maven from the left sidebar
  4. Configure project settings:
    • Name: Your project name (e.g., "file-reader-app")
    • Location: Project directory path
    • JDK: Select Java 11 or higher
  5. Expand Advanced Settings and configure Maven coordinates:
    • GroupId: com.example (reverse domain notation)
    • ArtifactId: file-reader-app (project identifier)
    • Version: 1.0-SNAPSHOT (development version)
  6. Click Create
Pro Tip: IntelliJ IDEA Community Edition includes full Maven support. You don't need the Ultimate version for Maven functionality.

Method 2: Import Existing Maven Project

  1. Click File → Open
  2. Navigate to the folder containing pom.xml
  3. Select the folder (or pom.xml directly) and click OK
  4. IntelliJ will detect Maven and import the project automatically

3. Understanding pom.xml

The pom.xml (Project Object Model) is the heart of a Maven project. It defines project configuration, dependencies, and build settings.

Basic pom.xml Structure

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <!-- Project Coordinates -->
    <groupId>com.example</groupId>
    <artifactId>file-reader-app</artifactId>
    <version>1.0-SNAPSHOT</version>

    <!-- Project Properties -->
    <properties>
        <maven.compiler.source>11</maven.compiler.source>
        <maven.compiler.target>11</maven.compiler.target>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    </properties>

    <!-- Dependencies -->
    <dependencies>
        <!-- Dependencies go here -->
    </dependencies>

    <!-- Build Configuration -->
    <build>
        <plugins>
            <!-- Build plugins go here -->
        </plugins>
    </build>
</project>

Key Elements Explained

Element Description Example
<groupId> Organization/company identifier (reverse domain) com.example
<artifactId> Project name/identifier file-reader-app
<version> Project version 1.0-SNAPSHOT
<properties> Configuration variables Java version, encoding
<dependencies> External libraries JUnit, Selenium, etc.
<build> Build configuration and plugins Compiler, test runners

4. Managing Dependencies

Dependencies are external libraries your project needs. Maven downloads them automatically from central repositories.

Adding a Dependency

To add a library (e.g., JUnit 5 for testing), add it to the <dependencies> section:

<dependencies>
    <!-- JUnit 5 for testing -->
    <dependency>
        <groupId>org.junit.jupiter</groupId>
        <artifactId>junit-jupiter-api</artifactId>
        <version>5.10.0</version>
        <scope>test</scope>
    </dependency>

    <!-- Selenium WebDriver for browser automation -->
    <dependency>
        <groupId>org.seleniumhq.selenium</groupId>
        <artifactId>selenium-java</artifactId>
        <version>4.15.0</version>
    </dependency>

    <!-- Apache POI for Excel file operations -->
    <dependency>
        <groupId>org.apache.poi</groupId>
        <artifactId>poi-ooxml</artifactId>
        <version>5.2.3</version>
    </dependency>

    <!-- JSON processing -->
    <dependency>
        <groupId>org.json</groupId>
        <artifactId>json</artifactId>
        <version>20230227</version>
    </dependency>
</dependencies>

Finding Dependencies on Maven Central

  1. Visit mvnrepository.com
  2. Search for the library (e.g., "junit jupiter")
  3. Click on the library name
  4. Select the desired version
  5. Copy the Maven dependency XML snippet
  6. Paste it into your pom.xml <dependencies> section

Dependency Scopes

Scope Description When to Use
compile Available in all phases (default) Production code libraries
test Only for testing phase JUnit, TestNG, Mockito
provided Provided by runtime environment Servlet API, JEE libraries
runtime Not needed for compilation JDBC drivers
After Adding Dependencies: IntelliJ may prompt you to reload the Maven project. Click the Maven refresh icon in the Maven tool window, or click "Load Maven Changes" popup.

5. Maven Project Structure

Maven enforces a standard directory layout that's consistent across all Maven projects:

project-root/
├── pom.xml                          (Project configuration)
├── src/
│   ├── main/
│   │   ├── java/                    (Production source code)
│   │   │   └── com/example/app/
│   │   │       ├── Main.java
│   │   │       ├── FileReader.java
│   │   │       └── ...
│   │   └── resources/               (Configuration files, properties)
│   │       ├── application.properties
│   │       └── log4j2.xml
│   └── test/
│       ├── java/                    (Test source code)
│       │   └── com/example/app/
│       │       ├── MainTest.java
│       │       └── FileReaderTest.java
│       └── resources/               (Test resources)
│           └── test-data.csv
└── target/                          (Generated files - never commit)
    ├── classes/                     (Compiled .class files)
    ├── test-classes/                (Compiled test .class files)
    └── project-name-1.0-SNAPSHOT.jar (Packaged JAR file)
Important: The target/ folder contains generated files and should be excluded from version control (add to .gitignore). Maven recreates it during builds.

6. Maven Lifecycle & Commands

Maven has three built-in build lifecycles: default (build), clean (cleanup), and site (documentation). The default lifecycle has several phases:

Common Maven Lifecycle Phases

Phase Description What It Does
clean Remove generated files Deletes target/ directory
validate Check project is correct Validates pom.xml
compile Compile source code Compiles src/main/java/ → target/classes/
test Run unit tests Runs JUnit/TestNG tests
package Create JAR/WAR file Packages compiled code into JAR
install Install to local repository Copies JAR to ~/.m2/repository
deploy Deploy to remote repository Uploads to artifact repository
Phase Execution: When you run a phase, Maven executes all preceding phases. For example, mvn package runs: validate → compile → test → package.

Most Common Commands

# Clean and compile project
mvn clean compile

# Run all tests
mvn test

# Build JAR file (without tests)
mvn clean package -DskipTests

# Build JAR file (with tests)
mvn clean package

# Install to local Maven repository
mvn clean install

# Run specific test class
mvn test -Dtest=FileReaderTest

# Run specific test method
mvn test -Dtest=FileReaderTest#testReadTxtFile

7. Using Maven Tool Window in IntelliJ

IntelliJ IDEA provides a dedicated Maven tool window for easy project management.

Opening the Maven Tool Window

  • Method 1: Click View → Tool Windows → Maven
  • Method 2: Click the Maven icon on the right sidebar
  • Method 3: Press Ctrl+Shift+A (Windows/Linux) or Cmd+Shift+A (Mac), then type "Maven"

Maven Tool Window Features

Project Structure

  • Expandable tree showing your project and sub-modules
  • View all dependencies (with transitive dependencies)
  • See plugin configurations

Lifecycle

Execute Maven phases with a double-click:

  • clean - Remove target directory
  • validate - Validate project
  • compile - Compile source code
  • test - Run unit tests
  • package - Create JAR file
  • verify - Run integration tests
  • install - Install to local repository
  • deploy - Deploy to remote repository

Plugins

View and execute plugin goals (e.g., compiler:compile, surefire:test)

Toolbar Actions

Icon Action Description
Reload All Maven Projects Refresh dependencies and configuration
Execute Maven Goal Run custom Maven commands
Toggle Skip Tests Skip test execution during builds
Show Dependencies Visualize dependency tree
Download Sources/Docs Download library source code
Pro Tip: Right-click on any Maven phase to create a Run Configuration for quick access. This adds it to the top toolbar for one-click execution.

8. Practical Example: Complete pom.xml for QA Automation

Here's a complete pom.xml configured for a Selenium test automation project with all essential dependencies:

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.example</groupId>
    <artifactId>qa-automation-framework</artifactId>
    <version>1.0-SNAPSHOT</version>

    <properties>
        <maven.compiler.source>11</maven.compiler.source>
        <maven.compiler.target>11</maven.compiler.target>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
        <selenium.version>4.15.0</selenium.version>
        <junit.version>5.10.0</junit.version>
    </properties>

    <dependencies>
        <!-- Selenium WebDriver -->
        <dependency>
            <groupId>org.seleniumhq.selenium</groupId>
            <artifactId>selenium-java</artifactId>
            <version>${selenium.version}</version>
        </dependency>

        <!-- JUnit 5 -->
        <dependency>
            <groupId>org.junit.jupiter</groupId>
            <artifactId>junit-jupiter-api</artifactId>
            <version>${junit.version}</version>
            <scope>test</scope>
        </dependency>
        <dependency>
            <groupId>org.junit.jupiter</groupId>
            <artifactId>junit-jupiter-engine</artifactId>
            <version>${junit.version}</version>
            <scope>test</scope>
        </dependency>

        <!-- TestNG (alternative to JUnit) -->
        <dependency>
            <groupId>org.testng</groupId>
            <artifactId>testng</artifactId>
            <version>7.8.0</version>
            <scope>test</scope>
        </dependency>

        <!-- Apache POI for Excel -->
        <dependency>
            <groupId>org.apache.poi</groupId>
            <artifactId>poi-ooxml</artifactId>
            <version>5.2.3</version>
        </dependency>

        <!-- ExtentReports -->
        <dependency>
            <groupId>com.aventstack</groupId>
            <artifactId>extentreports</artifactId>
            <version>5.1.1</version>
        </dependency>

        <!-- JSON Processing -->
        <dependency>
            <groupId>org.json</groupId>
            <artifactId>json</artifactId>
            <version>20230227</version>
        </dependency>

        <!-- Log4j2 for Logging -->
        <dependency>
            <groupId>org.apache.logging.log4j</groupId>
            <artifactId>log4j-core</artifactId>
            <version>2.20.0</version>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <!-- Maven Compiler Plugin -->
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-compiler-plugin</artifactId>
                <version>3.11.0</version>
                <configuration>
                    <source>11</source>
                    <target>11</target>
                </configuration>
            </plugin>

            <!-- Maven Surefire Plugin (for running tests) -->
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-surefire-plugin</artifactId>
                <version>3.1.2</version>
            </plugin>
        </plugins>
    </build>
</project>
Properties for Version Management: Notice how we use ${selenium.version} and ${junit.version}. This makes it easy to update library versions in one place.

9. Common Maven Issues & Solutions

Issue 1: Dependencies Not Downloading

Symptoms: Red imports, "Cannot resolve symbol" errors
Solutions:
  • Click Maven refresh icon in Maven tool window
  • Right-click project → Maven → Reload Project
  • File → Invalidate Caches → Invalidate and Restart
  • Check internet connection (Maven downloads from remote repositories)

Issue 2: "Maven Not Found" or JDK Issues

Solutions:
  • File → Project Structure → Project → Set correct JDK
  • File → Settings → Build, Execution, Deployment → Build Tools → Maven → Verify Maven home directory
  • IntelliJ bundles Maven - select "Use bundled Maven 3.x"

Issue 3: Old Dependencies Cached

Solution: Delete local repository cache
  • Navigate to ~/.m2/repository/ (Windows: C:\Users\YourName\.m2\repository\)
  • Delete the folder for the problematic dependency
  • Reload Maven project in IntelliJ

Issue 4: Tests Not Running

Solutions:
  • Ensure test classes are in src/test/java/
  • Test classes must end with "Test" (e.g., FileReaderTest.java)
  • Check "Toggle Skip Tests" button is not enabled
  • Verify JUnit/TestNG dependencies have <scope>test</scope>

10. Best Practices

✅ DO:
  • Use properties for version management
  • Keep dependencies up to date (but test before updating)
  • Use dependency scopes appropriately (test, compile, runtime)
  • Add .gitignore entry for target/ folder
  • Document custom Maven profiles in README
  • Run mvn clean before major builds
❌ DON'T:
  • Commit target/ folder to version control
  • Mix multiple testing frameworks (JUnit + TestNG) unnecessarily
  • Use outdated library versions with security vulnerabilities
  • Hardcode file paths in pom.xml
  • Ignore Maven warnings during builds
  • Skip tests without good reason
