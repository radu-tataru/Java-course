Instance vs Class Properties

Java classes can have two types of properties that serve different purposes:

Key Concept: Instance properties belong to each individual object, while static properties are shared across all objects of the class.
Type Keyword Shared or Unique? Example in Vehicle
Instance none Unique per object type, horsepower, comment
Class (Static) static Shared by all objects marca, count

Instance properties belong to each object. Every Vehicle can have a different type or horsepower.

Static properties belong to the class itself. All vehicles share the same marca ("Dacia") and the count of created vehicles.

Constructors: Object Initialization

Constructors are special methods used to create and initialize objects.

Basic Constructor

public Vehicle(String type, int horsepower, String comment) {
    this.type = type;
    this.horsepower = horsepower;
    this.comment = comment;
    count++; // increases the shared count
}
Constructor Overloading: You can have multiple constructors (called overloading) to allow different ways of creating objects.

Overloaded Constructor Example

public Vehicle(String type, int horsepower, String comment, String color) {
    this.type = type;
    this.horsepower = horsepower;
    this.comment = comment;
    this.color = color;
    count++;
}

Methods: Class vs Object

Method Type Keyword Called On Example
Object Method none an object Honk()
Class Method static the class sum()

Object Method Example

public void Honk() {
    System.out.println(this.type + " " + marca + " a claxonat!");
}

Called like: myCar.Honk();

Class Method Example

static void sum() {
    System.out.println(count);
}

Called like: Vehicle.sum();

Complete Demo

Here's how all these concepts work together:

Complete Vehicle Class

public class Vehicle {
    // Static properties (shared by all objects)
    static String marca = "Dacia";
    static int count = 0;

    // Instance properties (unique to each object)
    String type;
    int horsepower;
    String comment;
    String color;

    // Constructor
    public Vehicle(String type, int horsepower, String comment) {
        this.type = type;
        this.horsepower = horsepower;
        this.comment = comment;
        count++; // increases the shared count
    }

    // Overloaded constructor
    public Vehicle(String type, int horsepower, String comment, String color) {
        this.type = type;
        this.horsepower = horsepower;
        this.comment = comment;
        this.color = color;
        count++;
    }

    // Instance method
    public void Honk() {
        System.out.println(this.type + " " + marca + " a claxonat!");
    }

    // Static method
    static void sum() {
        System.out.println(count);
    }
}

Using the Vehicle Class

public class Main {
    public static void main(String[] args) {
        Vehicle car1 = new Vehicle("SUV", 150, "Good for families");
        Vehicle car2 = new Vehicle("Sedan", 120, "Fuel efficient");

        car1.Honk(); // SUV Dacia a claxonat!
        car2.Honk(); // Sedan Dacia a claxonat!

        Vehicle.sum(); // 2
    }
}

Summary

Use instance properties for data unique to each object.
Use static properties for shared data across all objects.
Constructors initialize objects and can be overloaded.
Object methods work with individual objects; static methods work with the class.
