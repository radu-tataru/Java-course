1. What are Generics?

Generics allow you to write code that works with different types while providing compile-time type safety. They enable you to create classes, interfaces, and methods that operate on objects of various types without sacrificing type checking.

The Problem Before Generics

// Before Java 5 (without generics)
ArrayList list = new ArrayList();
list.add("Hello");
list.add(123);
list.add(new Date());

String str = (String) list.get(0);  // Manual casting required
String num = (String) list.get(1);  // ❌ Runtime error: ClassCastException!

The Solution with Generics

// With generics (Java 5+)
ArrayList list = new ArrayList<>();
list.add("Hello");
list.add("World");
// list.add(123);  // ❌ Compile-time error - type safety!

String str = list.get(0);  // No casting needed

Benefits of Generics

  • Type Safety: Catch errors at compile-time instead of runtime
  • Eliminate Casting: No need for explicit type casting
  • Code Reusability: Write generic algorithms that work with different types
  • Better IDE Support: Improved code completion and error detection
Tip: Generics are checked at compile-time and erased at runtime (type erasure). This means generic type information is not available at runtime.

2. Using Generic Collections

ArrayList with Generics

// Creating a list of strings
ArrayList names = new ArrayList<>();
names.add("Alice");
names.add("Bob");
names.add("Charlie");

// Type-safe iteration
for (String name : names) {
    System.out.println(name.toUpperCase());  // No casting needed
}

// Creating a list of integers
ArrayList numbers = new ArrayList<>();
numbers.add(10);
numbers.add(20);
numbers.add(30);

int sum = 0;
for (Integer num : numbers) {
    sum += num;  // Auto-unboxing
}
System.out.println("Sum: " + sum);

HashMap with Generics

// Map of student IDs to names
HashMap students = new HashMap<>();
students.put(101, "Alice Smith");
students.put(102, "Bob Johnson");
students.put(103, "Charlie Brown");

// Type-safe retrieval
String name = students.get(101);  // Returns String, no casting
System.out.println("Student 101: " + name);

// Iterating over entries
for (Map.Entry entry : students.entrySet()) {
    System.out.println("ID: " + entry.getKey() + ", Name: " + entry.getValue());
}

Common Generic Collection Classes

Class Generic Type Description
ArrayList<E> E - element type Resizable array
LinkedList<E> E - element type Doubly-linked list
HashSet<E> E - element type Set with no duplicates
HashMap<K, V> K - key, V - value Key-value pairs
TreeMap<K, V> K - key, V - value Sorted key-value pairs
Warning: You cannot use primitive types with generics. Use wrapper classes instead: ArrayList<Integer> not ArrayList<int>.

3. Creating Generic Classes

Basic Generic Class

// Generic Box class that can hold any type
public class Box {
    private T content;

    public void set(T content) {
        this.content = content;
    }

    public T get() {
        return content;
    }

    public boolean isEmpty() {
        return content == null;
    }
}

// Usage
Box stringBox = new Box<>();
stringBox.set("Hello");
String value = stringBox.get();  // No casting needed

Box intBox = new Box<>();
intBox.set(42);
int number = intBox.get();

Generic Class with Multiple Type Parameters

public class Pair {
    private K key;
    private V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() {
        return key;
    }

    public V getValue() {
        return value;
    }

    @Override
    public String toString() {
        return "(" + key + ", " + value + ")";
    }
}

// Usage
Pair pair1 = new Pair<>("Age", 25);
System.out.println(pair1);  // (Age, 25)

Pair pair2 = new Pair<>(101, "Alice");
System.out.println(pair2);  // (101, Alice)

4. Generic Methods

Basic Generic Method

public class ArrayUtils {
    // Generic method to print array elements
    public static  void printArray(T[] array) {
        for (T element : array) {
            System.out.print(element + " ");
        }
        System.out.println();
    }
}

// Usage
String[] names = {"Alice", "Bob", "Charlie"};
Integer[] numbers = {1, 2, 3, 4, 5};

ArrayUtils.printArray(names);     // Alice Bob Charlie
ArrayUtils.printArray(numbers);   // 1 2 3 4 5

Generic Method with Return Type

public class Utils {
    // Returns the middle element of an array
    public static  T getMiddle(T[] array) {
        if (array == null || array.length == 0) {
            return null;
        }
        int middleIndex = array.length / 2;
        return array[middleIndex];
    }
}

// Usage
String[] words = {"apple", "banana", "cherry", "date"};
String middle = Utils.getMiddle(words);
System.out.println("Middle: " + middle);  // cherry

Integer[] nums = {10, 20, 30, 40, 50};
Integer midNum = Utils.getMiddle(nums);
System.out.println("Middle: " + midNum);  // 30

Generic Method in Non-Generic Class

public class Container {
    // Generic method in a regular class
    public  void display(T item) {
        System.out.println("Item: " + item);
        System.out.println("Type: " + item.getClass().getName());
    }
}

// Usage
Container container = new Container();
container.display("Hello");      // Item: Hello, Type: java.lang.String
container.display(42);           // Item: 42, Type: java.lang.Integer
container.display(3.14);         // Item: 3.14, Type: java.lang.Double
Tip: The type parameter <T> before the return type indicates it's a generic method. This is different from using a type parameter defined at the class level.

5. Bounded Type Parameters

Bounded type parameters restrict the types that can be used as generic arguments.

Upper Bounded Wildcards (extends)

// Only accepts Number or its subclasses (Integer, Double, etc.)
public class Calculator {
    private T number;

    public Calculator(T number) {
        this.number = number;
    }

    public double square() {
        return number.doubleValue() * number.doubleValue();
    }
}

// Usage
Calculator intCalc = new Calculator<>(5);
System.out.println("Square: " + intCalc.square());  // 25.0

Calculator doubleCalc = new Calculator<>(3.5);
System.out.println("Square: " + doubleCalc.square());  // 12.25

// Calculator strCalc = new Calculator<>("test");  // ❌ Compile error

Generic Method with Bounded Type

public class MathUtils {
    // Only accepts arrays of Number or subclasses
    public static  double sum(T[] numbers) {
        double total = 0;
        for (T num : numbers) {
            total += num.doubleValue();
        }
        return total;
    }
}

// Usage
Integer[] intArray = {1, 2, 3, 4, 5};
Double[] doubleArray = {1.5, 2.5, 3.5};

System.out.println("Sum: " + MathUtils.sum(intArray));     // 15.0
System.out.println("Sum: " + MathUtils.sum(doubleArray));  // 7.5

Multiple Bounds

// T must implement both Comparable and Serializable
public class SortedBox & Serializable> {

Note: When using multiple bounds, the class bound (if any) must come first, followed by interface bounds: <T extends Class & Interface1 & Interface2>

6. Wildcards

Unbounded Wildcard (?)

// Accepts a list of any type
public static void printList(List list) {
    for (Object item : list) {
        System.out.print(item + " ");
    }
    System.out.println();
}

// Usage
List strings = Arrays.asList("A", "B", "C");
List numbers = Arrays.asList(1, 2, 3);

printList(strings);  // A B C
printList(numbers);  // 1 2 3

Upper Bounded Wildcard (? extends Type)

// Accepts a list of Number or any subclass
public static double sumList(List list) {
    double sum = 0;
    for (Number num : list) {
        sum += num.doubleValue();
    }
    return sum;
}

// Usage
List integers = Arrays.asList(1, 2, 3);
List doubles = Arrays.asList(1.5, 2.5, 3.5);

System.out.println("Sum: " + sumList(integers));  // 6.0
System.out.println("Sum: " + sumList(doubles));   // 7.5
