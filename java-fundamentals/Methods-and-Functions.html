Method Basics

Methods are reusable blocks of code that perform specific tasks. They help organize code and avoid repetition.

Method Structure 
public class MethodDemo {
    
    // Method with return value
    public int calculateSum(int a, int b) {
        int result = a + b;
        return result;
    }
    
    // Method without return value (void)
    public void printMessage(String message) {
        System.out.println("Message: " + message);
    }
    
    // Method with no parameters
    public String getCurrentTime() {
        return java.time.LocalDateTime.now().toString();
    }
}
Method Anatomy:
access_modifier return_type methodName(parameters) { method_body }

Parameters and Return Types

Methods can accept input (parameters) and provide output (return values).

Parameter and Return Examples 
public class CalculatorDemo {
    
    // Method with multiple parameters and return value
    public double calculateArea(double length, double width) {
        return length * width;
    }
    
    // Method that modifies and returns a string
    public String formatName(String firstName, String lastName) {
        return firstName.toUpperCase() + ", " + lastName.toUpperCase();
    }
    
    // Method with boolean return type
    public boolean isEven(int number) {
        return number % 2 == 0;
    }
    
    // Method that validates input
    public String validateEmail(String email) {
        if (email == null || !email.contains("@")) {
            return "Invalid email format";
        }
        return "Email is valid";
    }
    
    public void demonstrateUsage() {
        // Calling methods and using return values
        double area = calculateArea(5.0, 3.0);
        String fullName = formatName("john", "doe");
        boolean evenCheck = isEven(10);
        String emailStatus = validateEmail("test@example.com");
        
        System.out.println("Area: " + area);
        System.out.println("Name: " + fullName);
        System.out.println("Is even: " + evenCheck);
        System.out.println("Email: " + emailStatus);
    }
}

Static vs Instance Methods

Static methods belong to the class, while instance methods belong to objects.

Static vs Instance Methods 
public class MethodTypesDemo {
    // Instance variable
    private String instanceName = "Instance Method Demo";
    
    // Static variable
    private static String staticName = "Static Method Demo";
    
    // Instance method - needs an object to be called
    public void instanceMethod() {
        System.out.println("Instance method called");
        System.out.println("Can access: " + instanceName);
        System.out.println("Can also access: " + staticName);
    }
    
    // Static method - can be called without creating an object
    public static void staticMethod() {
        System.out.println("Static method called");
        // System.out.println(instanceName); // ERROR! Cannot access instance variables
        System.out.println("Can access: " + staticName);
    }
    
    // Static utility method example
    public static int max(int a, int b) {
        return (a > b) ? a : b;
    }
    
    public static void main(String[] args) {
        // Calling static method directly
        staticMethod();
        int maximum = max(5, 10);
        
        // Calling instance method requires object creation
        MethodTypesDemo demo = new MethodTypesDemo();
        demo.instanceMethod();
    }
}
When to use Static: Use static methods for utility functions that don't need object data. Use instance methods when you need to work with object properties.

Method Overloading

You can have multiple methods with the same name but different parameters.

Method Overloading Examples 
public class OverloadingDemo {
    
    // Method overloading - same name, different parameters
    public void printInfo(String name) {
        System.out.println("Name: " + name);
    }
    
    public void printInfo(String name, int age) {
        System.out.println("Name: " + name + ", Age: " + age);
    }
    
    public void printInfo(String name, int age, String email) {
        System.out.println("Name: " + name + ", Age: " + age + ", Email: " + email);
    }
    
    // Overloaded calculation methods
    public int calculate(int a, int b) {
        return a + b;  // Addition for integers
    }
    
    public double calculate(double a, double b) {
        return a * b;  // Multiplication for doubles
    }
    
    public String calculate(String a, String b) {
        return a + " " + b;  // Concatenation for strings
    }
    
    public void demonstrateOverloading() {
        // Java automatically chooses the right method based on parameters
        printInfo("Alice");
        printInfo("Bob", 25);
        printInfo("Charlie", 30, "charlie@example.com");
        
        int intResult = calculate(5, 10);        // Calls int version
        double doubleResult = calculate(3.5, 2.0); // Calls double version
        String stringResult = calculate("Hello", "World"); // Calls String version
        
        System.out.println("Int result: " + intResult);
        System.out.println("Double result: " + doubleResult);
        System.out.println("String result: " + stringResult);
    }
}

Method Best Practices

Use descriptive names: calculateTotalPrice() instead of calc()
Keep methods focused: One method should do one thing well
Limit parameters: Avoid methods with too many parameters (max 3-4)
Use return values: Return results instead of just printing them

Common Method Patterns

Useful Method Patterns 
public class CommonPatterns {
    
    // Getter method pattern
    public String getName() {
        return this.name;
    }
    
    // Setter method pattern
    public void setName(String name) {
        this.name = name;
    }
    
    // Validation method pattern
    public boolean isValidAge(int age) {
        return age >= 0 && age <= 150;
    }
    
    // Utility method pattern
    public static String formatCurrency(double amount) {
        return String.format("$%.2f", amount);
    }
    
    // Factory method pattern
    public static User createUser(String name, String email) {
        User user = new User();
        user.setName(name);
        user.setEmail(email);
        return user;
    }
    
    // Method with error handling
    public int safeDivide(int dividend, int divisor) {
        if (divisor == 0) {
            System.err.println("Cannot divide by zero!");
            return 0;
        }
        return dividend / divisor;
    }
}

Key Takeaways

Method Structure: Access modifier, return type, name, parameters, and body
Static vs Instance: Static methods don't need objects, instance methods do
Method Overloading: Same name, different parameters for flexibility
Best Practices: Descriptive names, focused purpose, appropriate parameters
