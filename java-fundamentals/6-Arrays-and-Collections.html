Arrays Fundamentals

Arrays store multiple values of the same type in a fixed-size container.

Array Declaration and Initialization 
public class ArrayDemo {
    
    public void demonstrateArrays() {
        // Array declaration and initialization
        int[] numbers = {1, 2, 3, 4, 5};
        String[] names = {"Alice", "Bob", "Charlie"};
        
        // Alternative declaration syntax
        int[] moreNumbers = new int[5];  // Creates array of size 5 with default values (0)
        
        // Accessing array elements
        System.out.println("First number: " + numbers[0]);
        System.out.println("Last name: " + names[names.length - 1]);
        
        // Modifying array elements
        numbers[2] = 10;
        names[1] = "Robert";
        
        // Array length
        System.out.println("Numbers array length: " + numbers.length);
        
        // Iterating through arrays
        for (int i = 0; i < numbers.length; i++) {
            System.out.println("Index " + i + ": " + numbers[i]);
        }
        
        // Enhanced for loop (for-each)
        for (String name : names) {
            System.out.println("Name: " + name);
        }
    }
}
Array Limitations: Arrays have fixed size and cannot be resized after creation. Use collections for dynamic sizing.

ArrayList - Dynamic Arrays

ArrayList provides a resizable array implementation that can grow and shrink as needed.

ArrayList Operations 
import java.util.ArrayList;
import java.util.List;

public class ArrayListDemo {
    
    public void demonstrateArrayList() {
        // Creating ArrayList
        List fruits = new ArrayList<>();
        
        // Adding elements
        fruits.add("Apple");
        fruits.add("Banana");
        fruits.add("Orange");
        
        // Adding at specific index
        fruits.add(1, "Mango");  // Inserts at index 1
        
        // Accessing elements
        String firstFruit = fruits.get(0);
        System.out.println("First fruit: " + firstFruit);
        
        // Size of the list
        System.out.println("Number of fruits: " + fruits.size());
        
        // Checking if element exists
        if (fruits.contains("Apple")) {
            System.out.println("Apple is in the list");
        }
        
        // Removing elements
        fruits.remove("Banana");           // Remove by value
        fruits.remove(0);                  // Remove by index
        
        // Iterating through ArrayList
        for (String fruit : fruits) {
            System.out.println("Fruit: " + fruit);
        }
        
        // Using traditional for loop
        for (int i = 0; i < fruits.size(); i++) {
            System.out.println("Index " + i + ": " + fruits.get(i));
        }
        
        // Clear all elements
        fruits.clear();
        System.out.println("List is empty: " + fruits.isEmpty());
    }
}

HashMap - Key-Value Storage

HashMap stores data in key-value pairs, allowing fast lookups by key.

HashMap Operations 
import java.util.HashMap;
import java.util.Map;

public class HashMapDemo {
    
    public void demonstrateHashMap() {
        // Creating HashMap
        Map studentGrades = new HashMap<>();
        
        // Adding key-value pairs
        studentGrades.put("Alice", 95);
        studentGrades.put("Bob", 87);
        studentGrades.put("Charlie", 92);
        
        // Accessing values by key
        Integer aliceGrade = studentGrades.get("Alice");
        System.out.println("Alice's grade: " + aliceGrade);
        
        // Checking if key exists
        if (studentGrades.containsKey("Bob")) {
            System.out.println("Bob's grade: " + studentGrades.get("Bob"));
        }
        
        // Checking if value exists
        if (studentGrades.containsValue(92)) {
            System.out.println("Someone got 92%");
        }
        
        // Updating existing value
        studentGrades.put("Alice", 98);  // Updates Alice's grade
        
        // Getting value with default
        Integer unknownGrade = studentGrades.getOrDefault("David", 0);
        System.out.println("David's grade (default): " + unknownGrade);
        
        // Size of the map
        System.out.println("Number of students: " + studentGrades.size());
        
        // Iterating through HashMap
        for (String student : studentGrades.keySet()) {
            System.out.println(student + ": " + studentGrades.get(student));
        }
        
        // Alternative iteration using entrySet
        for (Map.Entry entry : studentGrades.entrySet()) {
            System.out.println(entry.getKey() + " scored " + entry.getValue());
        }
        
        // Removing elements
        studentGrades.remove("Bob");
        
        // Clear all elements
        studentGrades.clear();
    }
}

Choosing the Right Collection

Collection Type Use When Key Features
Array Fixed size, primitive types Fast access, memory efficient
ArrayList Dynamic size, ordered data Resizable, indexed access
HashMap Key-value pairs, fast lookups O(1) access, unique keys
LinkedList Frequent insertions/deletions Fast insertion, slower access

Practical Examples

Real-World Collection Usage 
import java.util.*;

public class CollectionExamples {
    
    // Managing a list of test cases
    public void manageTestCases() {
        List testCases = new ArrayList<>();
        testCases.add("Login with valid credentials");
        testCases.add("Login with invalid password");
        testCases.add("Login with empty fields");
        
        // Execute all test cases
        for (String testCase : testCases) {
            System.out.println("Executing: " + testCase);
        }
    }
    
    // Storing test results
    public void storeTestResults() {
        Map testResults = new HashMap<>();
        testResults.put("TC001", "PASS");
        testResults.put("TC002", "FAIL");
        testResults.put("TC003", "PASS");
        
        // Generate report
        int passCount = 0;
        for (String result : testResults.values()) {
            if ("PASS".equals(result)) {
                passCount++;
            }
        }
        
        System.out.println("Passed: " + passCount + "/" + testResults.size());
    }
    
    // Managing user data
    public void manageUserData() {
        List> users = new ArrayList<>();
        
        // Create user records
        Map user1 = new HashMap<>();
        user1.put("name", "John Doe");
        user1.put("email", "john@example.com");
        user1.put("role", "tester");
        
        Map user2 = new HashMap<>();
        user2.put("name", "Jane Smith");
        user2.put("email", "jane@example.com");
        user2.put("role", "admin");
        
        users.add(user1);
        users.add(user2);
        
        // Process user data
        for (Map user : users) {
            System.out.println("Processing user: " + user.get("name"));
        }
    }
}

Collection Best Practices

Use interfaces: List list = new ArrayList<>();
Initialize with capacity: new ArrayList<>(100) for large collections
Use enhanced for-loops: More readable than index-based loops
Check for null/empty: Always validate before operations

Key Takeaways

Arrays: Fixed size, fast access, good for primitive types
ArrayList: Dynamic size, ordered data, indexed access
HashMap: Key-value pairs, fast lookups, unique keys
Iteration: Enhanced for-loops are preferred for readability
