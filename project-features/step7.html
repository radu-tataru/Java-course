What is Page Object Model?

The Page Object Model (POM) is a design pattern that creates an object repository for storing web elements. It helps reduce code duplication and improves test maintenance by:

Benefits

  • Separates test logic from page-specific code
  • Makes tests more readable and maintainable
  • Reduces code duplication across test methods
  • Provides a single place to update element locators
  • Enables reusable page components

Considerations

  • Initial setup requires more planning
  • Additional abstraction layer to understand
  • Need to maintain page object classes
  • May be overkill for very simple test scenarios

Key Code Examples

Base Page Class

public class BasePage {
    protected WebDriver driver;
    protected WebDriverWait wait;
    
    public BasePage(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(10));
        PageFactory.initElements(driver, this);
    }
    
    protected void takeScreenshot(String testName) {
        // Screenshot implementation
    }
    
    protected boolean isElementDisplayed(WebElement element) {
        try {
            return element.isDisplayed();
        } catch (Exception e) {
            return false;
        }
    }
}

Specific Page Object

public class GitHubHomePage extends BasePage {
    @FindBy(css = "h1")
    private WebElement mainHeading;
    
    @FindBy(css = "a[href='/signup']")
    private WebElement signUpButton;
    
    public GitHubHomePage(WebDriver driver) {
        super(driver);
    }
    
    public String getMainHeading() {
        wait.until(ExpectedConditions.visibilityOf(mainHeading));
        return mainHeading.getText();
    }
    
    public boolean isSignUpButtonVisible() {
        return isElementDisplayed(signUpButton);
    }
    
    public void verifyPageLoaded() {
        wait.until(ExpectedConditions.titleContains("GitHub"));
        takeScreenshot("github_home_loaded");
    }
}

Test Implementation

public void testGitHubHomePage() {
    driver.get("https://github.com");
    GitHubHomePage gitHubPage = new GitHubHomePage(driver);
    
    gitHubPage.verifyPageLoaded();
    
    String heading = gitHubPage.getMainHeading();
    Logger.getInstance().log("GitHub heading: " + heading);
    
    boolean signUpVisible = gitHubPage.isSignUpButtonVisible();
    Logger.getInstance().log("Sign up button visible: " + signUpVisible);
    
    // Enhanced logging with assertions
    if (heading.toLowerCase().contains("github")) {
        Logger.getInstance().log("✅ GitHub page heading verification passed");
    } else {
        Logger.getInstance().log("❌ GitHub page heading verification failed");
    }
}

Professional Testing Insight

Page Object Model Benefits: This pattern is industry-standard in professional QA automation. It separates the "how" (element location and interaction) from the "what" (test logic), making your tests more maintainable and readable. When page layouts change, you only need to update the page object classes, not every test method.

Key Takeaways

Page Object Pattern: Separates page structure from test logic for better maintainability
Element Location: @FindBy annotations provide clean, maintainable element location strategies
Test Reliability: WebDriverWait and proper element handling increase test stability
Code Reusability: Page objects can be reused across multiple test scenarios
